问题描述

有一摞烙饼，因为一只手端着盘子，所以只能用另外一只手来给烙饼排序，将烙饼由大到小排好序。

算法思路

这个问题虽然看似麻烦，但是很多普通算法的思想都可以解决。

（1）冒泡排序思想

从顶层开始，每次并入一个烙饼，上一次的比较，如果更大则不用翻转，更小的话则将上一层翻转到此层，这样的话保证了每次并入的一层都是最大的。

一次自上而下的循环，可以将最大的那个烙饼放到最下面。再循环一次，可以将第二大的烙饼拍好，以此类推，可以得到整个排好序的烙饼。每次排放一个烙饼需要翻转两次，最多翻转的次数为n(n-1)。

（2）插入排序思想
从顶层开始，将已经并入的烙饼排序，成为一个子序列，每次并入一个烙饼，找到需要插入的位置，然后翻转，操作比较复杂。

如子序列{1,2,4,5,3}，这时3在最下面，插入位置在2和4之间，需要对2翻转，得到{2,1,4,5,3}，再对5翻转，得到{5,4,1,2,3}，然后对3翻转，得到{3,2,1,4,5}，再对3翻转得到{1,2,3,4,5}就完成了。

上述过程需要操作4次，但是如果并入的烙饼需要放到头部或尾部，操作就很少了。自上而下到最后一层，可以保证整个序列有序。最多翻转次数为4(n-2)+2。

  1. public static int[] sort(int[] arr) {
  2.     int len = arr.length;
  4.     //自上而下迭代
  5.     for (int i = len - 2; i >= 0; i--) {
  7.         //如果当前烙饼大于上一层，则已经有序，不用处理
  8.         if (arr[i] > arr[i + 1])
  9.             continue;
  11.         //如果当前烙饼小于顶层，则直接替换到最上面
  12.         if (arr[i] < arr[len - 1]) {
  13.             reverse(arr, i + 1, len);
  14.             reverse(arr, i, len);
  15.             continue;
  16.         }
  18.         //在中间位置的处理
  19.         int j = i + 1;
  21.         //查找插入位置
  22.         while (j != len - 1 && arr[i] < arr[j]) j++;
  24.         //reverse是翻转数组，并指定起始位置和终点位置
  25.         reverse(arr, j, len);
  26.         reverse(arr, i + 1, len);
  27.         reverse(arr, i, len);
  28.         reverse(arr, j - 1, len);
  29.     }
  30.     return arr;
  31. }

（3）选择排序

这也是最简单的，每次找到最大的烙饼，翻转两次，放到最下面，最后整个序列有序，翻转次数最多为2(n-1)。

（4）分支限界

只有遍历完所有的情况，挑选其中最优的。但是在搜索的过程中可以添加减支函数，减少搜索的次数。关键是递归函数的编写。
